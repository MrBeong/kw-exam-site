핵심 슬라이드로 외우는 구조

출처: 강의자료 10._회귀_및_분류문제의_성능평가_지표_설명.pdf, p.20.

  • 회귀 평가는 error 크기, 분류 평가는 confusion matrix 기반 비율로 정리한다.
  • Metric은 문제 목적에 맞춰 고른다. 불균형 데이터에서 accuracy만 믿으면 위험하다.
  • 시험에서는 metric 이름과 분자/분모를 정확히 연결해야 한다.

출처: 강의자료 10._회귀_및_분류문제의_성능평가_지표_설명.pdf, p.22.

  • Classification metric은 TP, FP, FN, TN을 정확히 놓고 시작한다.
  • Accuracy는 전체 중 맞춘 비율, precision은 positive라고 예측한 것 중 진짜 positive 비율, recall은 실제 positive 중 잡아낸 비율이다.
  • F1 score는 precision과 recall의 조화평균이므로 둘 중 하나만 높으면 크게 올라가지 않는다.

출처: 강의자료 10._회귀_및_분류문제의_성능평가_지표_설명.pdf, p.23.

  • Sensitivity 또는 recall은 실제 positive 중 모델이 positive로 맞춘 비율이다.
  • 질병 진단처럼 놓치면 큰 문제가 되는 경우 recall이 중요하다.
  • 식은 \(\frac{TP}{TP+FN}\)이다.

출처: 강의자료 10._회귀_및_분류문제의_성능평가_지표_설명.pdf, p.24.

  • Precision은 positive라고 예측한 것 중 실제 positive 비율이다.
  • False positive를 줄이는 것이 중요한 문제에서 precision이 중요하다.
  • 식은 \(\frac{TP}{TP+FP}\)이다.

출처: 강의자료 10._회귀_및_분류문제의_성능평가_지표_설명.pdf, p.29.

  • ROC는 threshold 변화에 따른 TPR과 FPR의 관계를 본다.
  • AUC가 클수록 positive와 negative를 잘 구분한다.
  • ROC는 threshold 하나가 아니라 전체 threshold 성능을 요약한다.

출처: 강의자료 10._회귀_및_분류문제의_성능평가_지표_설명.pdf, p.36.

  • K-fold cross validation은 데이터를 K개 fold로 나누고 validation fold를 바꿔가며 평가한다.
  • 한 번의 train/test split보다 성능 추정이 안정적이다.
  • Stratified K-fold는 classification에서 class 비율을 fold마다 유지한다.

출처: 강의자료 9._Ensemble_II_updated_20250429.pdf, p.3.

  • Bias는 모델 평균 예측과 참값 사이의 체계적 차이다.
  • Variance는 학습데이터가 바뀔 때 모델 예측이 흔들리는 정도다.
  • Ensemble은 여러 모델을 조합해 특히 variance를 줄이는 데 자주 쓰인다.

출처: 강의자료 9._Ensemble_II_updated_20250429.pdf, p.9.

  • Ensemble이 효과를 내려면 base learner들이 어느 정도 서로 다른 오류를 내야 한다.
  • Bootstrapping, random subspace, 서로 다른 알고리즘 조합이 diversity를 만든다.
  • 같은 모델을 완전히 똑같은 데이터로 반복하면 ensemble 효과가 작다.

출처: 강의자료 9._Ensemble_II_updated_20250429.pdf, p.11.

  • Random Forest는 bootstrapped sample과 random feature subset으로 여러 decision tree를 만든다.
  • Classification은 voting, regression은 averaging으로 예측한다.
  • OOB sample로 일반화 성능과 feature importance를 추정할 수 있다.

출처: 강의자료 9._Ensemble_II_updated_20250429.pdf, p.25.

  • AdaBoost는 이전 learner가 틀린 sample의 weight를 키워 다음 learner가 더 집중하게 한다.
  • Weak learner를 순차적으로 더해 strong learner를 만든다.
  • Bagging은 병렬/variance 감소, boosting은 순차/error correction으로 구분한다.

출처: 강의자료 9._Ensemble_II_updated_20250429.pdf, p.45.

  • XGBoost는 gradient boosting tree를 효율화하고 regularization을 포함한 모델이다.
  • 목적함수는 loss와 regularization term을 함께 최소화한다.
  • Missing value 처리, shrinkage, column subsampling 같은 실용 기능도 중요하다.

출처: 강의자료 9._Ensemble_II_updated_20250429.pdf, p.55.

  • LightGBM은 GOSS와 EFB로 학습 속도와 메모리 효율을 높인다.
  • Leaf-wise growth를 사용해 빠르게 loss를 줄이지만 과적합 주의가 필요하다.
  • 큰 데이터, 고차원 sparse feature에서 강점이 있다.

출처: 강의자료 9._Ensemble_II_updated_20250429.pdf, p.59.

  • CatBoost는 categorical feature 처리에 강하고 ordered target statistics를 사용한다.
  • Ordered 방식은 target leakage와 prediction shift를 줄이기 위한 장치다.
  • 퀴즈에서 CatBoost의 핵심 장점으로 자주 확인된 포인트다.

출처: 강의자료 9._Ensemble_II_updated_20250429.pdf, p.87.

  • Bagging은 같은 계열 모델을 병렬로 학습해 averaging/voting한다.
  • Boosting은 순차적으로 약한 학습기를 보완한다.
  • Stacking은 여러 base model의 예측을 meta model이 다시 학습한다.

출처: 강의자료 11._XAI_I.pdf, p.30.

  • Pre-model은 EDA처럼 모델 전 데이터 설명이다.
  • In-model은 decision tree, attention처럼 모델 내부에 설명 단서가 있는 경우다.
  • Post-hoc은 학습된 black-box 모델을 별도 방법으로 설명한다.

출처: 강의자료 11._XAI_I.pdf, p.31.

  • Intrinsic interpretability는 모델 자체가 설명 가능하도록 설계된 경우다.
  • Post-hoc interpretability는 예측 모델을 만든 뒤 별도 설명 방법을 붙인다.
  • 복잡한 모델일수록 post-hoc XAI가 중요해진다.

출처: 강의자료 11._XAI_I.pdf, p.32.

  • Decision tree와 linear model은 해석이 쉽지만 복잡한 패턴 표현이 약할 수 있다.
  • Deep learning은 예측력이 높을 수 있지만 black-box 성격이 강해 XAI가 필요하다.
  • 기말에서는 “모델 성능이 좋다”와 “설명 가능하다”를 같은 말로 보면 안 된다.

출처: 강의자료 11._XAI_I.pdf, p.42.

  • Model-agnostic method는 모델 종류와 상관없이 입력과 출력 반응을 이용해 설명한다.
  • PDP, ICE, permutation importance, LIME, SHAP을 이 범주로 묶어 외운다.
  • 핵심은 “모델 내부 구조를 몰라도 설명을 시도한다”이다.

출처: 강의자료 12._XAI_II.pdf, p.7.

  • PDP는 특정 feature 값을 변화시키고 나머지 feature를 marginalize해 평균 예측 변화를 본다.
  • 평균 효과를 보므로 global interpretation에 가깝다.
  • Feature 간 강한 상관이 있으면 비현실적인 조합이 생길 수 있다.

출처: 강의자료 12._XAI_II.pdf, p.26.

  • PDP는 특정 feature 변화가 평균 예측값에 주는 영향을 본다.
  • ICE는 sample별 곡선을 모두 그려서 개별 관측치의 반응 차이를 본다.
  • PDP는 global interpretation, ICE는 local pattern까지 더 잘 드러낸다.

출처: 강의자료 12._XAI_II.pdf, p.27.

  • Centered ICE는 sample별 시작점 차이를 제거하고 feature 변화에 따른 반응 형태를 비교한다.
  • 절대 예측값보다 변화 패턴을 보고 싶을 때 유용하다.
  • Sample 수가 많으면 곡선이 겹쳐 해석이 어려울 수 있다.

출처: 강의자료 12._XAI_II.pdf, p.38.

  • Morris screening은 한 번에 하나의 변수만 변화시키고 나머지는 고정해 민감도를 본다.
  • 평균 효과 \(\mu\)가 크면 평균적으로 중요한 변수이고, 표준편차 \(\sigma\)가 크면 변수 효과가 상황에 따라 크게 흔들린다.
  • \(\mu\)와 \(\sigma\)가 모두 작은 변수는 모델 예측에 영향이 작다고 해석한다.

출처: 강의자료 12._XAI_II.pdf, p.43.

  • 특정 feature를 shuffle했을 때 성능이 크게 떨어지면 중요한 feature다.
  • 모델 내부 구조와 무관하게 적용 가능하다.
  • Feature 간 상관이 강하면 중요도가 왜곡될 수 있다.

출처: 강의자료 13._XAI_III.pdf, p.16.

  • LIME은 특정 sample 주변을 perturbation하고 local surrogate model을 학습한다.
  • Local explanation이므로 “이 한 sample이 왜 이렇게 예측됐는가”를 설명한다.
  • 주변성 weighting과 단순 설명 모델이 핵심이다.

출처: 강의자료 14._XAI_IV.pdf, p.13.

  • SHAP은 Shapley value를 이용해 feature 기여도를 공정하게 배분한다.
  • 모든 feature subset에서의 marginal contribution을 평균하는 개념이다.
  • Additivity, consistency, local accuracy가 핵심 성질이다.

출처: 강의자료 14._XAI_IV.pdf, p.22.

  • SHAP은 feature가 예측값을 baseline에서 얼마나 밀어 올리거나 내렸는지 기여도로 해석한다.
  • Summary plot은 feature importance와 feature value 방향성을 동시에 보여준다.
  • 빨간 점과 파란 점의 위치를 함께 봐야 “값이 클수록 예측을 올리는지/내리는지”를 말할 수 있다.

출처: 강의자료 15._인공지능_최적화_및_메타휴리스틱스.pdf, p.20.

  • Training loss만 낮추면 overfitting이 생길 수 있다.
  • Regularization은 큰 계수나 복잡한 모델을 penalty로 억제한다.
  • \(\lambda\)가 너무 크면 underfitting, 너무 작으면 overfitting 위험이 있다.

출처: 강의자료 15._인공지능_최적화_및_메타휴리스틱스.pdf, p.56.

  • Batch gradient descent는 전체 데이터를 사용한다.
  • SGD는 sample 하나씩, mini-batch는 일부 묶음으로 gradient를 계산한다.
  • Mini-batch는 계산 효율과 gradient 안정성 사이의 절충이다.

출처: 강의자료 15._인공지능_최적화_및_메타휴리스틱스.pdf, p.70.

  • Gradient descent는 목적함수 \(J(w)\)가 감소하는 방향으로 \(w\leftarrow w-\eta\nabla J(w)\)를 반복한다.
  • Learning rate가 너무 크면 overshooting, 너무 작으면 수렴 지연이 생긴다.
  • Batch, SGD, mini-batch는 gradient를 계산하는 데이터 범위가 다르다.

출처: 강의자료 15._인공지능_최적화_및_메타휴리스틱스.pdf, p.72.

  • 초기 파라미터 설정, gradient 계산, learning rate만큼 이동, 수렴까지 반복 순서다.
  • 업데이트 식은 \(\theta\leftarrow\theta-\eta\nabla J(\theta)\)다.
  • 목적함수 최소화에서는 gradient 반대방향으로 움직인다.

출처: 강의자료 15._인공지능_최적화_및_메타휴리스틱스.pdf, p.86.

  • Grid search는 지정한 모든 조합을 체계적으로 탐색한다.
  • Random search는 임의 조합을 뽑아 탐색하므로 중요한 hyperparameter가 일부일 때 효율적일 수 있다.
  • Hyperparameter optimization은 validation 성능 기준으로 선택한다.

기계학습 최적화 핵심

개념 시험용 핵심
최적화 문제 목적함수 \(J(\theta)\)를 최소화하거나 최대화하는 파라미터 \(\theta\) 조합 탐색
기계학습 최적화 학습데이터의 loss를 줄이는 파라미터 \(W,b\)를 찾는 training phase
순수 최적화와 차이 순수 최적화는 목적함수 값 자체가 목표지만, 기계학습은 학습 loss 감소가 unseen data 성능 향상으로 이어져야 함
ERM Unknown data-generating distribution 대신 학습데이터의 empirical risk를 최소화하는 접근
과적합 학습데이터 loss만 과도하게 줄여 test/generalization 성능이 나빠지는 문제
정규화 loss에 penalty term을 추가해 계수가 과도하게 커지는 것을 억제
\(\lambda\) regularization strength. 너무 크면 underfitting, 너무 작으면 overfitting 위험
convex function 지역 최소해가 전역 최소해와 연결되기 쉬워 최적화가 상대적으로 쉬움
non-convex function local minima, saddle point, cliffs 등으로 최적화가 어려움
least squares 회귀에서 squared error를 최소화해 계수 추정
MLE 관측 데이터가 가장 그럴듯해지는 파라미터를 찾는 maximum likelihood 관점
MAP likelihood에 prior를 함께 반영해 posterior probability가 최대가 되는 파라미터를 찾는 관점
Bayesian estimator 파라미터를 하나의 점으로만 고정하지 않고 posterior distribution 전체를 고려하는 관점
logistic regression 최적화 log-likelihood를 최대화해야 하며 명시해가 어려워 gradient descent 등 수치 최적화 사용
EM algorithm latent variable 또는 missing value 때문에 log likelihood 계산이 어려울 때 E-step과 M-step을 반복
full batch 전체 데이터를 한 번에 이용해 gradient를 계산. 안정적이지만 큰 데이터/GPU 메모리 제약에서는 비현실적일 수 있음
online/SGD 샘플 1개마다 업데이트. 빠르고 즉각적이지만 업데이트가 noisy할 수 있음
minibatch full batch와 online의 중간. sample size가 클 때 현실적인 학습 방식이며 보통 \(8,16,32,64,128\)처럼 2의 거듭제곱 크기를 자주 사용
learning rate \(\eta\) 업데이트 폭. 너무 크면 overshooting, 너무 작으면 수렴이 느리거나 local minima에 갇힐 수 있음
learning rate decay 학습 후반부에 loss가 튀거나 진동할 때 learning rate를 줄여 수렴을 안정화하는 전략
gradient descent \(w \leftarrow w-\eta\nabla J(w)\) 형태로 목적함수가 감소하는 방향으로 이동
Hessian matrix 목적함수를 파라미터들에 대해 두 번 편미분한 2차 미분 행렬. 곡률 정보를 담지만 파라미터가 많으면 계산량이 커짐

momentum 이전 이동 방향을 반영해 local optimum 탈출을 도울 수 있지만 지나치면 좋은 local optimum도 지나칠 수 있음

gradient clipping cliffs처럼 gradient가 폭발적으로 커지는 구간에서 gradient 방향은 유지하되 norm을 threshold 이하로 제한
early stopping validation 성능을 모니터링해 과적합 전에 학습 종료
parameter 모델이 학습으로 추정하는 값. 예: neural network의 weight, bias

hyperparameter optimization \(\lambda\), hidden layer 수, node 수, activation 등 사용자가 정하는 값의 최적 조합 탐색
grid search 모든 격자 조합을 탐색. 차원이 커지면 계산량 폭증
random search 임의 조합 탐색. grid보다 효율적일 수 있으나 최적 보장 없음
Bayesian optimization Gaussian process 같은 surrogate model과 acquisition function으로 다음 탐색 지점을 고르는 hyperparameter 탐색 방식
heuristic 시간/자원 제약에서 충분히 좋은 approximate solution을 찾는 규칙 기반 접근

trajectory-based 하나의 해를 이동시키며 탐색. hill climbing, tabu search, simulated annealing 등

simulated annealing 나쁜 해도 확률적으로 받아들이며 온도 \(T\)를 낮춰 전역 탐색에서 국소 탐색으로 전환
genetic algorithm parameter set을 chromosome으로 보고 fitness 평가, mutation, crossover, replacement를 반복

최적화 단원 시험 함정

  • 최적화 = training loss 최소화라고만 쓰면 부족하다. 기계학습에서는 training data의 loss가 줄어도 unseen/test data 성능이 좋아진다는 보장이 없으므로 generalization을 같이 봐야 한다.
  • ERM은 학습데이터에서의 empirical risk를 줄이는 방식이지만, 이 자체가 과적합 방지책은 아니다.
  • 작은 데이터에서 과적합을 줄이는 대표 전략은 label data 추가 확보, regularization, early stopping, cross-validation, Bayesian approach, data augmentation 등이다.
  • Global minimum은 전체 탐색공간에서 가장 좋은 해이고, local minimum은 주변에서는 좋지만 전체 최선은 아닐 수 있다.
  • Convex에서는 최적화가 상대적으로 쉽지만, 신경망처럼 non-convex한 문제에서는 ill-conditioning, local minima, saddle point, cliffs가 문제가 된다.
  • Batch size는 한 번 업데이트할 때 쓰는 샘플 수이고, epoch는 전체 데이터를 몇 번 반복했는지이다.
  • Full batch, mini-batch, online/SGD를 구분해야 한다. 전체 데이터를 한 번에 쓰면 full batch, 일부 묶음이면 mini-batch, 샘플 1개마다 업데이트하면 online/SGD에 가깝다.
  • Learning rate가 크면 빠르지만 overshooting 위험이 있고, 작으면 안정적이지만 느리며 local minima에 갇힐 수 있다.
  • loss curve가 초반에는 잘 내려가다가 후반에 튀면 learning rate decay를 고려할 수 있다.
  • Early stopping은 training 성능이 아니라 validation 성능을 모니터링해 과적합 전에 멈추는 전략이다.
  • Hyperparameter tuning을 validation set에 반복하면 validation set에도 과적합될 수 있으므로, k-fold cross-validation 같은 방식이 언급된다.
  • Least squares는 회귀에서 squared error를 최소화하는 방식이고, Gaussian noise 가정에서는 MLE와 연결된다.
  • Probability는 파라미터가 주어졌을 때 데이터/사건이 나올 확률이고, likelihood는 관측 데이터가 주어졌을 때 파라미터가 얼마나 그럴듯한지를 보는 함수이다.
  • MLE에서 여러 sample의 likelihood는 곱 형태가 되므로, log likelihood를 쓰면 곱을 합으로 바꿔 계산과 최적화를 쉽게 만든다.
  • MLE, MAP, Bayesian estimator는 모두 파라미터 추정 관점이지만, MLE는 likelihood, MAP는 prior까지 반영한 posterior mode, Bayesian estimator는 posterior distribution/expectation 관점이다.
  • MLE는 학습데이터가 적으면 관측된 샘플에 민감하게 치우칠 수 있다. 이때 prior를 반영한 MAP나 Bayesian approach가 과적합 완화 관점에서 언급된다.
  • EM은 E-step에서 log likelihood의 기댓값을 계산하고, M-step에서 이를 최대화하는 절차를 수렴할 때까지 반복한다.
  • Parameter와 hyperparameter를 구분해야 한다. Weight/bias는 학습되는 parameter이고, learning rate, batch size, epoch, hidden layer 수, node 수, activation, \(\lambda\)는 사용자가 정하거나 validation으로 고르는 hyperparameter이다.
  • Grid search는 격자 전체 탐색이라 차원이 늘면 계산량이 폭증한다. Random search는 전체 보장은 약하지만 제한된 자원에서 효율적일 수 있다.
  • 일반적으로 heuristic/metaheuristic은 제한 시간 내 충분히 좋은 근사해를 찾는 전략이다. 단, Kruskal/Dijkstra처럼 특정 문제에서는 heuristic하게 보이는 알고리즘도 최적해를 보장할 수 있으므로 “heuristic은 절대 최적해를 못 찾는다”는 선지는 틀릴 수 있다.

1-2. Bias-Variance와 No Free Lunch

개념 핵심 함정
Bias 모델 예측의 평균이 실제 정답에서 체계적으로 벗어나는 정도 단순 모델은 보통 high bias
Variance training data가 바뀔 때 모델 예측이 흔들리는 정도 복잡 모델은 보통 high variance
낮은 복잡도 모델 linear regression 같은 단순 모델은 일반적으로 high bias, low variance 항상 나쁘다는 뜻은 아님
높은 복잡도 모델 deep neural network 같은 모델은 상대적으로 low bias, high variance 경향 데이터/정규화/학습법에 따라 일반화 가능
Optimal complexity bias와 variance의 단순 교차점이 아니라 전체 expected error가 최소가 되는 지점 “교차점에서 최적” 선지 주의
Bagging 효과 여러 고분산 모델을 평균/투표해 variance 감소 bias 감소가 주목적이라는 설명은 부정확
Boosting 효과 약한 모델을 순차 보완해 bias 감소 경향 learner가 너무 강하면 boosting 취지와 안 맞을 수 있음
No Free Lunch 모든 데이터/문제에서 항상 우월한 단일 알고리즘은 없음 딥러닝이나 앙상블이 NFL을 해결했다는 선지는 틀림

추가 암기 포인트

Random Forest와 앙상블 함정 요약

  • Random Forest는 bagging/bootstrap과 random subspace method가 핵심이다.
  • Random Forest는 단순 decision tree 집합이 아니며, pruning을 강하게 적용하는 것이 이론적 핵심은 아님.
  • Bagging은 variance 감소, Boosting은 bias 감소 경향으로 연결해서 설명.
  • Random Forest에서 loss regularization을 반복 학습에 추가한다는 설명은 부적절하며, 앙상블 구조 자체가 regularization 효과를 가짐.
  • Random Forest 정확도 향상 문항에서는 학습 데이터 추가 확보가 가장 안정적인 정답으로 설명됨. 변수 선택법, 변동성 방지를 위한 데이터 축소, pretraining/fine-tuning식 미세튜닝 선지는 부정확.
  • Bias-Variance 문항에서 낮은 복잡도 선형 모델은 high bias/low variance, 신경망은 상대적으로 low bias/high variance 경향이라고 설명.
  • Optimal model complexity는 bias와 variance의 단순 교차점이 아니라 전체 예측 오류가 최소화되는 지점으로 봐야 함.
  • No Free Lunch theorem은 어떤 알고리즘도 모든 데이터에서 항상 우월할 수 없다는 의미이며, 딥러닝이나 앙상블이 이를 해결했다는 선지는 틀림.
  • Boosting 그림 해석에서는 병렬 bagging이 아니라 sequential/iterative 보완 구조를 읽어야 하며, majority voting은 training이 아니라 inference 결합 방식이라고 강조.
  • CatBoost Ordered Target Statistics는 target encoding의 target leakage와 prediction shift를 완화하기 위한 방법으로 설명.
  • Heterogeneous stacking의 장점은 서로 다른 알고리즘들이 같은 실수를 할 가능성이 낮아 상호 보완을 기대할 수 있다는 점.
  • AI 예측은 예언과 달리 정량적 validation과 정확도 평가가 필수라고 설명.

2. 오늘 수업 핵심 정리

Random Forest와 앙상블

  • Random Forest는 supervised learning이다.
  • 핵심은 decision tree 여러 개가 아니라 bootstrap sampling과 random subspace/feature subsampling으로 다양성을 만드는 구조다.
  • Bagging은 고분산 모델을 여러 개 만들어 aggregate하면서 variance를 줄이는 데 초점이 있다.
  • Boosting은 앞 learner의 오답이나 residual을 뒤 learner가 보완해 bias를 줄이는 데 초점이 있다.
  • Bagging 쪽 weak learner는 깊고 강하게 만들어도 aggregate로 안정화하는 방향이고, Boosting 쪽 weak learner는 너무 강하면 순차 보완 구조의 장점이 줄어든다.

Bias-Variance와 No Free Lunch

  • 낮은 복잡도 모델, 예: 선형모델은 일반적으로 high bias, low variance.
  • 높은 복잡도 모델, 예: 인공신경망은 상대적으로 low bias, high variance 경향.
  • 최적 모델 복잡도는 bias와 variance가 교차하는 지점이 아니라 전체 예측오류가 최소가 되는 지점이다.
  • No Free Lunch theorem은 어떤 알고리즘도 모든 데이터와 문제에서 항상 우월할 수 없다는 뜻이다.
  • 딥러닝이 최근 강력하더라도 No Free Lunch theorem을 무효화한 것은 아니다.

성능평가지표

  • AI 예측은 예언과 달리 validation과 정량적 평가가 필수다.
  • 회귀 기본 지표: MAE, MSE, RMSE, MAPE, MASE, R square.
  • 분류 기본 지표: accuracy, recall/sensitivity/TPR, specificity/TNR, precision/PPV, F1, ROC/AUC.
  • Confusion matrix는 라이브러리마다 actual/predicted 축 배치가 다를 수 있으므로 식을 계산하기 전에 행과 열을 먼저 확인해야 한다.
  • Precision과 Recall은 trade-off 관계로 함께 보는 것이 일반적이며, F1 score는 둘의 조화평균이다.
  • ROC curve는 x축이 1 - specificity 또는 FPR, y축이 sensitivity 또는 TPR이다.

교차검증

  • Train set에 과적합되는 문제를 막기 위해 validation set을 둔다.
  • 그런데 validation set으로 하이퍼파라미터를 계속 고르면 validation set에도 과적합될 수 있다.
  • Cross-validation은 validation fold를 바꿔가며 여러 번 평가해 특정 split에 대한 의존을 줄이는 방법이다.
  • 딥러닝에서는 시간/비용 문제와 다른 일반화 전략 때문에 cross-validation을 일반적으로 많이 하지 않는다.

XAI 도입

  • 정확도만 높이는 것이 전부가 아니다. 최신 모델은 틀려도 confidence를 매우 높게 내는 overconfidence 문제가 생길 수 있다.
  • Calibration은 confidence와 실제 정답률이 더 잘 맞도록 조정하는 관점이다.
  • XAI는 어떤 예측이 왜 나왔는지 사람이 이해할 수 있게 설명하는 과정이다.
  • 법률, 금융, 의학, 로봇/제어처럼 예측이 의사결정이나 행동으로 연결되는 분야에서 특히 중요하다.
  • 잘못된 이미지 분류 예시처럼, 설명은 모델이 어디를 보고 판단했는지 보여주고 데이터 편향이나 학습 문제를 찾는 데 쓰인다.

5. 주관식 답안 틀

Random Forest를 설명하시오.

Random Forest는 여러 decision tree를 결합하는 supervised ensemble이다. 단순히 tree를 많이 모으는 것이 아니라 bootstrap sampling으로 sample 다양성을 만들고, random subspace 또는 feature subsampling으로 feature 선택의 무작위성을 더해 tree 간 diversity를 확보한다. Bagging 계열이므로 주된 효과는 variance 감소다.

Precision과 Recall을 비교하시오.

Recall은 실제 positive 중 모델이 positive로 잡은 비율이고 \(\frac{TP}{TP+FN}\)이다. Precision은 positive라고 예측한 것 중 실제 positive인 비율이고 \(\frac{TP}{TP+FP}\)이다. 둘은 trade-off가 생길 수 있어 함께 확인하며, F1 score는 precision과 recall의 조화평균이다.

Cross-validation의 목적을 설명하시오.

Train set에 과적합되지 않도록 validation set을 두지만, 하이퍼파라미터 선택을 반복하면 validation set에도 과적합될 수 있다. Cross-validation은 validation fold를 여러 번 바꿔 평가함으로써 특정 validation split에 대한 의존을 줄이고 일반화 성능 추정을 안정화한다.

XAI가 정확도와 별개로 필요한 이유를 설명하시오.

정확도가 높아도 모델이 왜 그런 예측을 했는지 모르면 법률, 금융, 의료, 제어처럼 의사결정이 중요한 분야에서 신뢰와 책임을 확보하기 어렵다. 또한 최신 모델은 틀린 예측에도 높은 confidence를 줄 수 있어 overconfidence와 calibration 문제가 생긴다. XAI는 특정 예측의 근거를 사람이 이해할 수 있게 보여주고, 데이터 편향이나 모델 오류를 찾는 데 도움을 준다.

XAI I-II

0. 오늘 강의 한줄 요약

XAI는 모델이 왜 그런 예측을 했는지 설명하기 위한 방법이다. 오늘은 설명을 언제/어떻게 붙일지, \(f(x)\) 예측 모델과 g(x) 설명 모델의 차이, global/local 및 model-specific/agnostic 구분, 그리고 PDP, ICE, Morris, Permutation Importance를 다뤘다.

1. XAI가 필요한 이유

AI 모델이 아무리 성능이 좋아도, 실제 현장에서는 “왜 그렇게 판단했는가”를 설명해야 한다.

예:

  • 은행 대출 거절: 고객에게 어떤 요인 때문에 거절됐는지 설명해야 함.
  • 의료 AI: 환자에게 어떤 feature 때문에 위험하다고 판단했는지 설명해야 함.
  • 자동화 시스템: 잘못된 행동이 발생했을 때 책임과 원인을 추적해야 함.

주의:

XAI는 내부 parameter 작동을 전부 완벽히 해부한다는 뜻이 아니다.
입력 feature와 예측 결과 사이의 관계를 사람이 이해할 수 있게 설명하는 것이다.

2. Black box를 여는 시점

시점 의미 예
모델 전 데이터 자체를 먼저 설명 EDA, visualization, label별 분포 확인
모델 중 해석 가능한 구조를 모델에 내장 linear regression, decision tree, attention score
모델 후 학습된 모델을 별도 방법으로 설명 PDP, ICE, Permutation, LIME, SHAP

데이터 자체의 intrinsic pattern을 먼저 이해하는 EDA가 중요하다.

3. Intrinsic vs Post-hoc

구분 핵심 예
Intrinsic interpretability 모델 자체가 해석 가능 linear regression, decision tree
Post-hoc interpretability 예측 모델 학습 후 별도 설명 PDP, ICE, Morris, Permutation, LIME, SHAP

시험용 문장:

Post-hoc interpretation은 black-box model f를 먼저 학습한 뒤, 별도의 설명 방법이나 설명 모델 g로 f의 예측을 해석하는 방식이다.

4. \(f(x)\)와 g(x)

기호 의미
\(f(x)\) prediction model, black-box 예측 모델
g(x) explanation model, 설명 모델

\(f(x)\)는 예측을 잘하는 것이 목적이고, g(x)는 그 예측을 사람이 이해할 수 있도록 설명하는 것이 목적이다.

5. 설명 가능성 수준

수준 특징 장점 단점
High interpretability linear, monotonic 설명이 명확 복잡한 local pattern 놓침
Medium interpretability 일부 비선형 반영 균형 완전 단순하지 않음
Low interpretability 복잡한 패턴까지 근사 세부 패턴 설명 가능 설명이 길고 모호해짐

핵심 trade-off:

간단한 설명은 명쾌하지만 부정확할 수 있고,
정교한 설명은 정확하지만 사람이 이해하기 어려울 수 있다.

6. Global/Local, Model-specific/Model-agnostic

축 구분 의미
설명 대상 Global 모델 전체/평균 경향 설명
설명 대상 Local 특정 관측치 하나의 예측 이유 설명
모델 의존성 Model-specific 특정 모델 구조에 의존
모델 의존성 Model-agnostic 모델 종류와 무관하게 적용

예:

  • Random Forest feature importance: global explanation.
  • 특정 환자가 왜 위험군인지 설명: local explanation.
  • Decision tree rule: model-specific.
  • PDP/ICE/Permutation/LIME: model-agnostic 성격.

7. 모델별 해석 특징

모델 해석 장점 한계
Linear regression beta coefficient로 방향/크기 설명 가능 비선형 패턴 약함
Decision tree rule extraction 가능 bias/overfitting 가능
Random Forest 평균적 feature importance 가능 개별 샘플 local explanation은 약함
Deep learning 예측 성능 높음 기본적으로 black-box, 별도 XAI 필요

8. PDP

PDP는 관심 feature를 변화시킬 때 모델 예측이 평균적으로 어떻게 변하는지 본다.

관심 feature: x_s
나머지 feature: x_c

나머지 feature는 marginalization하여 없애고, \(x_s\) 변화에 따른 평균 예측 변화를 본다.

시험 답안

PDP는 특정 feature의 marginal effect를 보여주는 global model-agnostic explanation이다.

장단점

장점 단점
직관적이고 평균적 경향을 명확히 보여줌 개별 샘플 차이와 feature interaction을 가릴 수 있음

9. ICE

ICE는 PDP를 sample별로 쪼개서 보여준다.

PDP: 평균 curve 1개
ICE: instance별 curve 여러 개

시험 답안

ICE는 각 관측치마다 관심 feature 변화에 따른 prediction 변화를 보여주므로 heterogeneous relation을 확인할 수 있다.

장단점

장점 단점
sample-wise pattern 확인 가능 샘플이 많으면 overcrowded되어 해석 어려움

10. Centered ICE

ICE curve가 서로 다른 시작값을 가지면, feature 변화 효과보다 절대 수준 차이가 더 커 보일 수 있다.

Centered ICE는 기준점에서 curve를 맞춰 상대적 변화량을 본다.

절대 예측값 차이보다 feature 변화에 따른 변화량을 보고 싶을 때 사용

11. Morris Sensitivity Analysis

Morris는 one-at-a-time 방식의 sensitivity analysis다.

나머지 feature 고정
관심 feature 하나만 변화
prediction 변화량 측정
값 의미
\(\mu\) feature의 평균적 영향력
\(\sigma\) 영향이 얼마나 흔들리는지, interaction/nonlinearity 가능성

해석:

  • \(\mu\)와 \(\sigma\)가 모두 작음: 거의 무시 가능.
  • \(\mu\)가 큼: 평균적으로 중요한 feature.
  • \(\sigma\)가 큼: feature effect가 상황에 따라 크게 달라짐.

12. Permutation Importance

Permutation importance는 feature 값을 섞어서 해당 feature의 정보를 깨뜨리고, 성능 하락을 본다.

feature shuffle 후 성능이 많이 떨어지면 중요한 feature

장점:

  • 모델 종류와 무관하게 적용 가능.
  • 직관적.

주의:

  • 상관된 feature가 있으면 중요도 해석이 왜곡될 수 있음.
  • 보통 global importance로 많이 쓰지만, 특정 샘플 기준으로 변형하면 local interpretation도 가능하다.

13. Correlation vs Mutual Information

개념 의미 한계/장점
Correlation coefficient 선형 관계 측정 비선형 관계를 놓칠 수 있음
Mutual Information 확률분포 기반 관계 측정 선형/비선형 관계를 더 일반적으로 포착
KL divergence 분포 차이 측정 MI 설명에 연결되는 정보이론 개념

시험 함정:

correlation이 낮다고 해서 관계가 전혀 없다고 단정하면 안 된다.
선형 관계가 약할 뿐, 비선형 관계가 있을 수 있다.

15. XAI 한계

핵심 한계:

XAI는 인과적 root cause를 완벽하게 찾아주는 마법이 아니다.
대부분은 어떤 feature가 prediction에 크게 기여했는지를 보여주는 수준이다.

따라서 XAI 결과는 domain knowledge와 함께 해석해야 한다.

16. 바로 외울 단답

  1. \(f(x)\)는 예측 모델, g(x)는 설명 모델이다.
  2. Intrinsic interpretability는 모델 자체가 해석 가능한 경우다.
  3. Post-hoc interpretability는 모델 학습 후 별도 방법으로 설명하는 방식이다.
  4. Global explanation은 전체 평균 경향, local explanation은 특정 샘플 예측 이유다.
  5. Model-agnostic method는 모델 종류와 무관하게 적용 가능한 설명 방법이다.
  6. PDP는 특정 feature의 평균적 marginal effect를 보여준다.
  7. ICE는 각 instance별 prediction curve를 보여준다.
  8. Centered ICE는 시작점을 맞춰 상대적 변화량을 본다.
  9. Morris는 one-at-a-time sensitivity analysis다.
  10. Permutation importance는 feature를 섞었을 때 성능 하락량으로 중요도를 본다.
  11. Correlation은 선형 관계만 주로 본다.
  12. Mutual information은 확률분포 기반으로 비선형 관계도 포착할 수 있다.

18. 의사코드 작성형 답안 틀

의사코드는 변수명보다 초기화 → 반복 → 평가/업데이트 → 출력 순서를 맞추는 것이 핵심이다.

AdaBoost

Input: training data, weak learner, number of learners T
Initialize sample weights D_i = 1/n
For t = 1 to T:
    Train weak learner h_t using weights D
    Compute weighted error epsilon_t
    Compute learner weight alpha_t
    Increase weights of misclassified samples
    Decrease weights of correctly classified samples
    Normalize D
Output: weighted majority vote of weak learners

핵심 암기:

\[ \alpha_t=\frac{1}{2}\ln\frac{1-\epsilon_t}{\epsilon_t} \]

Gradient Boosting

Initialize model F_0
For m = 1 to M:
    Compute residual or negative gradient of loss
    Fit weak learner h_m to that residual
    Find step size or learner weight
    Update F_m = F_{m-1} + learning_rate × h_m
Output: final additive model F_M

핵심 암기:

AdaBoost는 sample weight를 바꾸고,
Gradient Boosting은 loss의 residual/negative gradient를 줄이는 learner를 더한다.

Permutation Importance

Train model
Measure baseline validation score
For each feature j:
    Copy validation data
    Shuffle only feature j
    Measure new validation score
    Importance_j = baseline score - shuffled score
Rank features by importance

PDP

Choose target feature s
Choose grid values for feature s
For each grid value z:
    Replace feature s of all samples with z
    Predict using trained model
    Average predictions
Plot z versus averaged prediction

ICE

Choose target feature s
For each sample i:
    For each grid value z:
        Replace feature s of sample i with z
        Predict using trained model
    Plot the prediction curve for sample i

LIME

Choose target sample x
Generate perturbed samples around x
Get black-box model predictions for perturbed samples
Weight samples by proximity to x
Fit simple interpretable model g
Explain x using coefficients or rules of g

SHAP

Treat features as players in a cooperative game
For each feature:
    Compare model output with and without that feature over feature coalitions
    Average marginal contribution as Shapley value
Explain prediction as base value plus feature contributions

핵심 암기:

\[ f(x)=\phi_0+\sum_{j=1}^{p}\phi_j \]

Mini-batch Gradient Descent

Initialize parameters w
For each epoch:
    Shuffle training data
    Split data into mini-batches
    For each mini-batch:
        Compute average gradient on mini-batch
        Update w = w - learning_rate × gradient
Output trained parameters

Early Stopping

Initialize best validation score
For each epoch:
    Train model for one epoch
    Evaluate validation score
    If validation score improves:
        Save current model
        Reset patience counter
    Else:
        Increase patience counter
    Stop if patience counter reaches limit
Output best saved model

Grid Search와 Random Search

Grid search:
    Define fixed candidate values for each hyperparameter
    Evaluate every combination
    Choose best validation score

Random search:
    Define search distribution for each hyperparameter
    Randomly sample combinations within budget
    Choose best validation score

Genetic Algorithm

Initialize population of candidate solutions
Evaluate fitness of each candidate
Repeat:
    Select strong candidates
    Apply crossover
    Apply mutation
    Replace population
Output best candidate

Simulated Annealing

Initialize current solution and temperature T
Repeat:
    Generate neighbor solution
    If neighbor is better, accept it
    If neighbor is worse, accept it with probability depending on T
    Decrease T
Output best solution

핵심 암기:

초반에는 탐색을 넓게 하고, 온도가 낮아질수록 나쁜 해를 덜 받아들인다.

19. 그래프 해석 문제 답안 문장

  1. SHAP plot에서 x축 오른쪽은 예측값을 증가시키는 양의 기여, 왼쪽은 예측값을 감소시키는 음의 기여다.
  2. SHAP plot에서 색은 feature value의 크기다. 빨간색은 feature 값이 큰 sample, 파란색은 feature 값이 작은 sample이다.
  3. SHAP plot에서 빨간 점이 오른쪽에 많으면 feature 값이 클수록 positive class 또는 예측값을 증가시킨다.
  4. SHAP plot에서 빨간 점이 왼쪽에 많으면 feature 값이 클수록 positive class 또는 예측값을 감소시킨다.
  5. PDP에서 오른쪽으로 갈수록 y가 증가하면 feature 값 증가가 평균 prediction을 증가시킨다.
  6. ICE 선들이 많이 교차하면 sample마다 feature effect가 다르므로 interaction 또는 heterogeneous relation 가능성이 있다.
  7. Morris plot에서 \(\mu\)가 크면 평균적으로 중요하고, \(\sigma\)가 크면 효과가 일정하지 않다.
  8. ROC는 왼쪽 위에 가까울수록 좋고, AUC가 1에 가까울수록 좋다.
  9. PR curve는 class imbalance가 심할 때 positive class 성능을 보기 좋다.
  10. Permutation importance가 크다는 것은 해당 feature 정보를 깨뜨렸을 때 성능 하락이 크다는 뜻이다.
