영상AI생성모델 퀴즈 대비 통합 문제세트

1. Transformer, MHA, ViT 기본 문제

  1. 표준 multi-head attention에서 \(d_{\mathrm{model}}=512\), head 수 \(h=8\)이면 한 head의 \(d_k,d_v\)는?

  2. \(d_k=d_v=32\)

  3. \(d_k=d_v=64\)

  4. \(d_k=512,\ d_v=64\)

  5. \(d_k=64,\ d_v=512\)

정답: 2

풀이: 표준 MHA에서는 \(d_k=d_v=d_{\mathrm{model}}/h=512/8=64\)이다.

  1. 표준 MHA의 weight 수는 bias 제외 시 \(4d_{\mathrm{model}}^2\)이다.

정답: O

풀이: \(W^Q,W^K,W^V,W^O\) 네 projection weight를 세므로 \(4d^2\)이다.

  1. \(d=12\), \(h=3\), \(d_k=5\), \(d_v=4\)인 variant MHA의 weight 수를 구하라. Bias 제외.

정답: \(648\)

풀이: Variant MHA는 \(h\,d(2d_k+2d_v)=3\times12\times(10+8)=648\)이다.

  1. Padding mask와 causal mask의 설명으로 맞는 것은?

  2. Padding mask는 미래 token을 막고 causal mask는 PAD token을 막는다.

  3. 둘 다 학습되는 parameter이므로 parameter count에 포함한다.

  4. Padding mask는 PAD token을 무시하고 causal mask는 미래 token을 보지 못하게 한다.

  5. Causal mask는 encoder self-attention에서만 사용한다.

정답: 3

풀이: Padding mask는 PAD 위치를 attention에서 제외하고, causal mask는 autoregressive decoding에서 미래 token을 차단한다.

  1. Image가 \(224\times224\times3\), patch가 \(16\times16\)이면 ViT patch token 수는? Class token 제외.

정답: \(196\)

풀이: \((224/16)\times(224/16)=14\times14=196\)이다.

  1. Patch embedding projection weight shape으로 맞는 것은? Image channel \(C\), patch size \(P\), model 차원 \(d_{\mathrm{model}}\)이다.

  2. \(P^2C\times d_{\mathrm{model}}\)

  3. \(d_{\mathrm{model}}\times P\)

  4. \(C\times P\times d_{\mathrm{model}}\)

  5. \(N_{\mathrm{patch}}\times P^2C\)

정답: 1

풀이: Patch 하나를 flatten하면 \(P^2C\)차원이고 이를 \(d_{\mathrm{model}}\)로 projection한다.

  1. ViT에서 positional embedding이 필요한 이유를 쓰시오.

정답: Transformer는 patch token의 2D 위치를 자동으로 알지 못하므로 각 patch의 위치 정보를 더하기 위해 필요하다.

풀이: Self-attention은 token 집합의 관계를 보지만 이미지상의 공간 위치를 별도 입력 없이 알 수 없다.

2. Diffusion, DDPM, DDIM

  1. Diffusion model의 forward process 설명으로 맞는 것은?

  2. Noise에서 시작해 image를 바로 decode한다.

  3. 원본 image에 timestep별 noise를 점점 더한다.

  4. Text embedding만 학습하고 image는 쓰지 않는다.

  5. Classifier gradient로 class를 분류한다.

정답: 2

풀이: Forward process는 깨끗한 sample에 noise를 추가해 random noise에 가까운 상태로 만드는 과정이다.

  1. Diffusion reverse process에서 neural network가 흔히 예측하는 것은?

  2. 제거해야 할 noise

  3. Dataset의 caption 개수

  4. Patch token 수

  5. RGB channel 순서

정답: 1

풀이: DDPM 계열은 noisy sample \(x_t\)에 섞인 noise \(\epsilon\)을 예측하도록 학습하는 방식으로 이해하면 된다.

  1. Diffusion model에 timestep \(t\)를 넣는 이유를 쓰시오.

정답: Timestep마다 noise level이 다르므로 모델이 현재 입력에서 어느 정도의 noise를 제거해야 하는지 알게 하기 위해서다.

풀이: 같은 구조의 U-Net이라도 \(t\) 정보가 없으면 약한 noise와 강한 noise 상태를 구분하기 어렵다.

  1. DDPM의 practical 단점으로 가장 적절한 것은?

  2. Text prompt를 절대 사용할 수 없다.

  3. Sampling step이 많아 생성 속도가 느리다.

  4. Forward process가 학습되지 않는다.

  5. Image를 patch로 나눌 수 없다.

정답: 2

풀이: DDPM은 많은 timestep을 순차적으로 거쳐야 해서 sampling이 느리다.

  1. DDIM은 DDPM의 모든 timestep을 그대로 촘촘히 따라가므로 sampling 속도가 느려진다.

정답: X

풀이: DDIM은 일부 timestep을 건너뛰어 더 적은 step으로 sampling하는 것이 핵심이다.

  1. DDIM에서 timestep을 무작정 \(1,2,3,\dots,19,1000\)처럼 크게 건너뛰면 문제가 되는 이유는?

정답: 마지막 큰 jump에서 noise 분포의 variance가 과도하게 커져 Gaussian transition 가정이 깨지고 sampling이 불안정해질 수 있다.

풀이: Step 수를 줄이더라도 timestep 간격과 noise schedule이 분포를 유지할 정도로 설계되어야 한다.

3. CLIP과 조건 정보

  1. CLIP 학습의 핵심으로 맞는 것은?

  2. Image만 autoencoding한다.

  3. 맞는 image-text pair의 similarity를 키우고 틀린 pair의 similarity를 줄인다.

  4. Text token을 causal mask로만 생성한다.

  5. Classifier gradient를 diffusion step에 더한다.

정답: 2

풀이: CLIP은 image encoder와 text encoder를 contrastive learning으로 같은 의미 공간에 맞춘다.

  1. CLIP similarity matrix에서 diagonal이 의미하는 것은?

  2. 틀린 image-text pair

  3. 같은 image끼리의 pixel distance

  4. 정답 image-text pair

  5. U-Net skip connection

정답: 3

풀이: Batch 안에서 diagonal은 서로 맞는 image-text pair이고 off-diagonal은 negative pair다.

  1. CLIP에서 hard negative가 중요한 이유를 쓰시오.

정답: 서로 다른 pair라도 의미가 비슷해 similarity가 높게 나올 수 있으므로 모델이 더 세밀한 구분을 학습하게 만들기 때문이다.

풀이: 쉬운 pair만 있으면 diagonal만 크게 만드는 문제가 너무 쉬워져 표현 학습이 약해질 수 있다.

  1. Text-to-image diffusion에서 CLIP/text encoder의 출력은 생성 모델의 condition으로 쓰일 수 있다.

정답: O

풀이: Text embedding은 U-Net의 cross-attention 등으로 들어가 prompt 의미를 denoising 과정에 제공한다.

4. Classifier Guidance와 CFG

  1. Classifier guidance 설명으로 맞는 것은?

  2. 별도 classifier의 gradient로 원하는 class 방향을 준다.

  3. Classifier 없이 null condition만 사용한다.

  4. VAE decoder만 학습하는 방법이다.

  5. Timestep을 제거하는 방법이다.

정답: 1

풀이: Classifier guidance는 diffusion model 외에 classifier를 두고 그 gradient를 reverse process에 사용한다.

  1. Classifier-free guidance 설명으로 맞는 것은?

  2. 별도 classifier를 반드시 학습한다.

  3. Conditional prediction과 unconditional prediction을 섞어 condition 반영 강도를 조절한다.

  4. Text encoder를 사용하면 안 된다.

  5. Forward process에서만 사용된다.

정답: 2

풀이: CFG는 별도 classifier 없이 diffusion model의 conditional/unconditional 예측 차이를 이용한다.

  1. CFG가 classifier-free라고 불리는 이유를 쓰시오.

정답: Guidance를 위해 별도 classifier를 쓰지 않고 diffusion model 자체가 만든 conditional prediction과 unconditional prediction을 사용하기 때문이다.

풀이: Classifier guidance와 달리 별도 분류기의 gradient가 필요 없다.

  1. CFG에서 null condition을 학습하는 목적은?

  2. Patch 수를 줄이기 위해서

  3. Unconditional prediction을 얻기 위해서

  4. VAE decoder를 제거하기 위해서

  5. CLIP similarity matrix를 만들기 위해서

정답: 2

풀이: Sampling 때 condition 없는 예측과 condition 있는 예측을 비교해야 하므로 null condition 학습이 필요하다.

  1. CFG 대표 식에서 \(s\)가 의미하는 것은?

\[ \hat{\epsilon} =\epsilon_{\theta}(x_t,\varnothing,t) +s\left(\epsilon_{\theta}(x_t,c,t)-\epsilon_{\theta}(x_t,\varnothing,t)\right) \]

정답: Guidance scale

풀이: \(s\)가 클수록 condition 방향을 더 강하게 반영한다.

  1. CFG scale이 너무 크면 어떤 문제가 생길 수 있는가?

정답: Prompt 반영은 강해지지만 이미지가 과하게 조형화되고 자연스러움이나 다양성이 줄어들 수 있다.

풀이: 조건 방향을 너무 세게 밀면 unconditional prior와의 균형이 깨질 수 있다.

  1. CFG scale이 너무 작으면 prompt 영향이 약해져 unconditional 생성에 가까워질 수 있다.

정답: O

풀이: \(s\)가 작으면 conditional prediction과 unconditional prediction의 차이를 거의 밀어주지 않는다.

5. Stable Diffusion과 LDM 구조

  1. Stable Diffusion에서 diffusion이 주로 일어나는 공간은?

  2. Raw pixel space

  3. VAE latent space

  4. Text token vocabulary

  5. Classifier output space

정답: 2

풀이: Stable Diffusion은 VAE가 만든 latent map에서 denoising을 수행하는 latent diffusion 계열이다.

  1. Stable Diffusion 구조에서 VAE encoder의 역할은?

  2. Prompt를 token으로 나눈다.

  3. Image를 latent map으로 압축한다.

  4. Noise schedule을 삭제한다.

  5. 최종 class label을 분류한다.

정답: 2

풀이: VAE encoder는 image pixel을 더 작은 latent representation으로 보낸다.

  1. Stable Diffusion 구조에서 VAE decoder의 역할을 쓰시오.

정답: Denoising이 끝난 latent map을 다시 image로 복원한다.

풀이: Latent space에서 생성된 결과는 decoder를 지나야 사람이 보는 image가 된다.

  1. Stable Diffusion에서 U-Net이 받는 입력으로 가장 적절한 조합은?

  2. Noisy latent, timestep, text condition

  3. Text token만

  4. 최종 image와 정답 class만

  5. Patch index와 FFN hidden dimension만

정답: 1

풀이: U-Net은 \(z_t\), timestep embedding, text embedding condition을 이용해 noise를 예측한다.

  1. Stable Diffusion에서 cross-attention은 prompt 정보와 image latent feature를 연결하는 데 쓰인다.

정답: O

풀이: Text embedding이 key/value 역할을 하고 image latent feature가 prompt 의미를 참고하도록 만든다.

  1. Pixel-space diffusion보다 latent diffusion이 효율적인 이유는?

정답: 고해상도 pixel 전체가 아니라 압축된 latent map에서 denoising하므로 계산해야 할 공간이 작아지기 때문이다.

풀이: 예를 들어 \(512\times512\) image를 \(64\times64\) latent map처럼 줄이면 denoising 비용이 크게 감소한다.

  1. Stable Diffusion에서 latent channel은 반드시 RGB 3채널과 같아야 한다.

정답: X

풀이: Latent channel 수는 모델 설계에 따라 달라지며 4채널 이상이 될 수 있다.

  1. Stable Diffusion 전체 흐름으로 맞는 것은?

  2. Prompt → VAE decoder → classifier → image

  3. Prompt → text encoder → U-Net condition, latent denoising → VAE decoder → image

  4. Image → causal mask → CLIP 삭제 → image

  5. Patch embedding → MHA parameter count → image

정답: 2

풀이: Prompt는 text embedding이 되어 U-Net condition으로 들어가고, denoising된 latent는 VAE decoder로 image가 된다.

6. DiT, LoRA, fine-tuning

  1. DiT 설명으로 맞는 것은?

  2. Diffusion process 자체를 없애는 모델이다.

  3. Denoising backbone에 Transformer 계열 block을 쓰는 diffusion 구조다.

  4. CLIP의 text encoder만 의미한다.

  5. VAE decoder를 classifier로 바꾼 구조다.

정답: 2

풀이: DiT는 diffusion process보다 denoising network backbone을 Transformer화하는 관점으로 외운다.

  1. DiT는 U-Net과 무관하게 forward noise process를 completely 제거한다.

정답: X

풀이: DiT도 diffusion framework 안에서 denoising network 구조를 바꾸는 쪽에 가깝다.

  1. LoRA를 diffusion model에 쓰는 이유로 맞는 것은?

  2. 전체 U-Net을 처음부터 다시 학습하지 않고 작은 adapter만 학습하기 위해서

  3. Noise schedule을 없애기 위해서

  4. VAE latent를 RGB pixel로 고정하기 위해서

  5. Causal mask를 padding mask로 바꾸기 위해서

정답: 1

풀이: LoRA는 기존 큰 weight는 유지하고 작은 low-rank adapter를 학습해 style/domain adaptation 비용을 줄인다.

  1. 특정 style에 맞춘 image generation을 만들 때 LoRA가 유리한 이유를 쓰시오.

정답: 거대한 diffusion model 전체를 재학습하지 않고 작은 추가 weight만 학습해 원하는 style이나 domain을 반영할 수 있기 때문이다.

풀이: 비용과 시간이 큰 full fine-tuning 대신 lightweight fine-tuning을 수행하는 방식이다.

7. Conditional diffusion 퀴즈 보강

  1. Conditional diffusion model에서 condition으로 들어갈 수 있는 대표 입력은?

  2. text prompt embedding

  3. 항상 scalar learning rate만

  4. optimizer의 momentum만

  5. GPU memory address

정답: 1

풀이: text-to-image diffusion에서는 text encoder가 만든 text embedding이 denoising network의 condition으로 들어간다.

  1. Conditional diffusion에서 null condition을 함께 쓰는 이유로 가장 적절한 것은?

  2. condition이 없는 예측과 condition이 있는 예측을 조합해 guidance 강도를 조절하기 위해

  3. VAE decoder를 제거하기 위해

  4. diffusion timestep을 항상 0으로 만들기 위해

  5. image resolution을 무조건 줄이기 위해

정답: 1

풀이: classifier-free guidance는 unconditional prediction과 conditional prediction을 섞어 prompt 반영 강도를 조절한다.

  1. Guidance scale을 너무 크게 주면 생길 수 있는 현상으로 가장 적절한 것은?

  2. prompt 조건은 강하게 반영되지만 이미지가 과도하게 조형화되거나 artifact가 생길 수 있다.

  3. condition이 완전히 사라진다.

  4. noise가 학습 데이터로 변한다.

  5. LoRA rank가 자동으로 0이 된다.

정답: 1

풀이: guidance가 강하면 prompt 부합성은 올라갈 수 있지만 자연스러움과 다양성이 손상될 수 있다.

  1. 특정 작가풍이나 지브리풍처럼 style을 반영하고 싶을 때 전체 U-Net을 처음부터 다시 학습하지 않는 대표 방법은?

  2. LoRA 또는 lightweight fine-tuning

  3. Hough Transform

  4. Random forest

  5. Maxwell relation

정답: 1

풀이: LoRA는 큰 diffusion model은 유지하고 작은 추가 weight만 학습해 특정 style을 반영한다.
